# 透過サムネトリックメーカー

X (Twitter) で流行している「サムネイルでは一部しか見えないのに、クリックすると全体が現れる」透過PNGイラストを作る、ブラウザ完結の1枚HTMLツールです。

画像を読み込み、サムネイルで見せたい部分をブラシでなぞって、PNGを書き出すだけ。書き出したPNGはそのままXに投稿できます。

**▶ 公開ページ: https://komanushi.github.io/trick_thumbnail/**

## 使い方

### 起動

ブラウザで `index.html` を直接開くだけでも動きます。開発用サーバーを使う場合:

```bash
python3 serve.py        # http://127.0.0.1:9292/ (デフォルト)
python3 serve.py 3000   # ポート指定
```

依存ライブラリはありません(Python 標準ライブラリのみ)。

### 操作

1. 画像をドラッグ&ドロップ / 「画像を開く」/ Ctrl+V で読み込む(最初は全体が「隠す」状態)
2. サムネイルで見せたい部分をブラシでなぞる(赤いベールが消えた所が見える部分)
3. 「サムネ(TL)」タブでタイムラインでの見え方、「クリック後」タブで拡大表示の見え方を確認
4. 「PNGを書き出す」で `〇〇_trick.png` を保存し、Xに投稿

| 操作 | キー |
|---|---|
| ブラシ / 消しゴム | `B` / `E` |
| 選択範囲を反転 | `I` |
| ブラシサイズ | `[` / `]` |
| 元に戻す / やり直す | `Ctrl+Z` / `Ctrl+Shift+Z` |
| ズーム / 移動 | ホイール / `Space`+ドラッグ or 中ボタンドラッグ |

## 仕組み

実際にXで動作しているトリック画像(PNG-8・アルファ2値・1px市松模様)の解析結果に基づいています。

- **見せる部分**: 完全不透明・元の色のまま
- **隠す部分**: 1pxの市松模様で50%を完全透過に、残り50%を元の色の2倍の明るさにする
- **サムネイル**: Xはサムネ生成(縮小)時に半透明になったピクセルを完全透過に落とすため、市松領域(縮小後アルファ≒50%)は丸ごと消える。テーマの背景色が透けるだけなので、**ライト/ダークどちらのテーマでも隠れる**
- **クリック後**: 拡大表示は黒背景+市松がそのまま届くため、「2倍の色 × 50%」が平均されて元の色に復元される
- **ファイル形式**: X が無条件でPNGのまま保持する PNG-8(パレット+tRNS)で書き出す。エンコーダは自前実装(median cut 減色 + Floyd–Steinberg ディザ + CompressionStream)

## 制限事項

原理(Xの画像パイプライン)に由来するもので、どのツールでも回避できません。

1. **隠し部分は「元の色の2倍」までしか持ち上げられない**
   クリック後の復元は最大2倍まで。RGB値127以下のピクセルは完全に復元されますが、**128以上は約127で頭打ち**になり、本来より暗く浮き上がります(隠した純白はグレーになる)。暗色系のイラスト向きの手法です。
   緩和策を2つ実装しています:
   - **色相保持クリップ(常時ON)**: 頭打ちをチャンネル個別ではなくピクセル単位の倍率で行うため、白く濁らず色味を保ったまま暗くなります
   - **写真モード(任意)**: 画像全体にトーン圧縮(Reinhard型、255→127.5)をかけてからトリック加工します。全体が少し暗くなる代わりにクリップが一切発生せず、見せる部分と隠し部分の明るさの継ぎ目が消えます。明るい写真に有効です
2. **見せる部分の輪郭に白っぽいフチが出る**
   境界ピクセルは縮小後のアルファが50%を超えて生き残り、2倍明度の色で表示されるため。実際に流行している画像にも同じフチがあります。
3. **256色に減色される**
   PNG-8 のため。ディザで補っていますが、微妙なグラデーションは劣化します。実際に流行しているサンプル画像も同じく256色に減色されています。
4. **長辺4096pxまで**
   Xの画像上限。超える画像は読み込み時に自動縮小されます。
5. **Xの公式クライアント専用のトリック**
   Discord・他サイト・サードパーティクライアント・保存した画像をビューアで開いた場合などは、背景色次第で全部見えたり全部消えたりします。白背景では隠し部分が白っぽいゴーストとして薄く見えます。
6. **秘匿ではない**
   ただの透過PNGなので、画像を保存して黒背景に置けば誰でも隠し部分を見られます。見せたくないものを隠す用途には使えません。
7. **拡大表示で等倍以上にズームすると市松の網目が見える**
   通常の閲覧距離では平均されて元の絵に見えます。
8. **PCブラウザとXモバイルアプリで、クリック後の隠し部分の明るさが約1.46倍異なる**
   モバイルアプリの拡大ビューはリニア色空間(GPU)で市松を平均するため、sRGB空間で平均するPCブラウザより明るく復元されます(`2^(1-1/2.2)≈1.46倍`)。なぞった部分は透過を含まないため変化せず、合っていない側の環境では相対的に影(または逆に明るい浮き)として見えます。原理上、両方で同時に正確な復元はできないため、ツールバーの**「クリック後の基準」で PC / モバイルを選択**してください(隠し部分の倍率 2.0 / 1.37 を切り替えます)。明るい写真ほど差が目立ち、暗い画像では気になりません。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `index.html` | アプリ本体(HTML/CSS/JS すべて込み、外部依存なし) |
| `serve.py` | 開発用HTTPサーバー(キャッシュ無効、デフォルトポート9292) |
| `samples/` | 実際にXで動作しているトリック画像のサンプル(方式解析の元) |

## 開発メモ

- `window.__test` に `processTrick` / `encodePNG8` / `setImageForTest` を公開しており、Playwright 等からコアロジックを直接テストできます
- 加工は `processTrick()`(Bayer 4x4 ディザ、被覆率50%固定)、書き出しは `encodePNG8()` を参照
